# Event Project Management

## Events as Projects

Every event is a project with:
- Defined scope and objectives
- Fixed deadline (the event date)
- Budget constraints
- Multiple stakeholders
- Resource coordination
- Risk management needs

The difference: Events have an immovable deadline and the "product" is delivered live.

## Project Lifecycle

### Phase 1: Initiation (Week 1-2)
**Activities:**
- Client brief and discovery
- Feasibility assessment
- Initial scope definition
- Rough budget development
- Go/no-go decision
- Proposal and agreement

**Deliverables:**
- Signed agreement
- Project charter (event brief)
- High-level timeline
- Preliminary budget

### Phase 2: Planning (Ongoing until Week -4)
**Activities:**
- Detailed requirements gathering
- Venue selection and booking
- Vendor procurement
- Budget refinement
- Risk identification
- Run sheet development
- Stakeholder communication plan

**Deliverables:**
- Detailed event plan
- Confirmed vendors
- Final budget
- Risk register
- Draft run sheet

### Phase 3: Execution Preparation (Week -4 to -1)
**Activities:**
- Final confirmations
- Detailed run sheet completion
- Team briefings
- Site visits and walk-throughs
- Contingency finalisation
- Client sign-offs

**Deliverables:**
- Final run sheet
- Vendor briefing packs
- Team assignments
- Contingency plans
- Final client approval

### Phase 4: Delivery (Event Day/s)
**Activities:**
- Setup supervision
- Team coordination
- Run sheet execution
- Real-time problem solving
- Client liaison
- Pack down management

**Deliverables:**
- Successful event execution
- Issue log
- Initial feedback

### Phase 5: Closure (Week +1 to +2)
**Activities:**
- Vendor reconciliation
- Final invoicing
- Client feedback collection
- Team debrief
- Documentation and filing
- Lessons learned

**Deliverables:**
- Final invoice
- Client feedback
- Lessons learned document
- Project closure

## Stakeholder Management

### Primary Stakeholders
| Stakeholder | Interest | Engagement Level |
|-------------|----------|------------------|
| Client (decision maker) | Event success, budget | High - regular meetings |
| Client (day-of contact) | Smooth execution | High - event day focus |
| Venue manager | Facility use, reputation | Medium - key milestones |
| Key vendors | Deliver their component | Medium - as needed |

### Stakeholder Communication
- **Client**: Weekly updates minimum, more frequent near event
- **Vendors**: At booking, confirmation, and pre-event
- **Internal team**: Daily standups in final week
- **Casual staff**: Pre-event briefing only

### Managing Expectations
- Document all decisions and changes
- Confirm understanding in writing
- Flag risks and constraints early
- No surprises - communicate proactively

## Risk Management

### Common Event Risks

| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| Vendor no-show | Low | High | Backup vendors, confirmation calls |
| Weather (outdoor) | Medium | High | Wet weather plan, marquee backup |
| AV failure | Medium | High | Tech rehearsal, backup equipment |
| Guest numbers change | High | Medium | Flex catering, venue capacity buffer |
| Client late changes | High | Medium | Change process, cutoff dates |
| Staff shortage | Medium | Medium | Over-roster, backup pool |

### Risk Response Planning
- **Avoid**: Don't use unreliable vendors
- **Mitigate**: Tech rehearsals reduce AV risk
- **Transfer**: Insurance for major risks
- **Accept**: Minor guest number variations

### Contingency Planning
Every critical element needs a backup:
- Backup venue contact for emergencies
- Second-choice vendors on standby
- Spare equipment for key items
- Alternative program if speaker cancels

## Change Management

### Change Request Process
1. Change requested (client or team)
2. Impact assessment (time, cost, feasibility)
3. Options presented to client
4. Decision documented
5. Plan updated
6. Stakeholders notified

### Change Control Thresholds
- Minor (no cost/time impact): Coordinator approves
- Moderate (<$500 or minor timeline): Client approval required
- Major (>$500 or significant change): Formal change request

### Cutoff Dates
- Guest numbers: 7-14 days before event
- Menu changes: 14 days before event
- Major scope changes: 21+ days before event
- Run sheet changes: 48 hours before event
